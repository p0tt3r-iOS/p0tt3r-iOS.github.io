- RandomAccessCollection.distance(from:to:)

    ```swift
    func distance(from start: Self.Index, to end: Self.Index) -> Int
    ```

- NotificationCenter 동작 방식과 활용 방안에 대해 설명하시오.  
    - 동작 방식  
        - Notification Center는 특정 객체를 addObserver(_:) 메서드를 이용해 알림을 받을 수 있는 옵저버로 만든다.  
        - 객체가 옵저버로 추가되면 자신이 어떤 알림을 받아야 하는 지와 어떤 동작을 할 지 정한다.  
            (여러 알림을 받고 싶은 경우, addObserver(_:) 메서드를 여러 번 부를 수 있다.)  
        - 해당 객체는 알림이 오면, 정해진 동작을 수행한다.  

    - 활용 방안
        - UIKit에서 기본적으로 제공하는 Notification을 활용하여,  
            시스템 적인 변화 또는 라이프 사이클이 일어났을 때, 특정 동작을 실행할 수 있다.  
            (예시로 userDidTakeScreenshotNotification, willResignActiveNotification 등이 있음)  
        - 다른 뷰 객체에서 비동기적으로 실행한 작업이, 뷰 전환 후 완료되었을 때 특정 동작을 실행 시킬 수 있다.  
            예를 들어, 1번 뷰에서 다운로드를 클릭하고, 2번 뷰로 이동한 후에 완료되어도  
            Notification을 보내서 alert를 띄우는 등의 동작을 할 수 있다.
  
    - NotificationCenter을 아직 사용해본 적은 없지만, 유용할 것 같다는 생각과 동시에  
        동일한  기능을 하는 다른 객체 / 디자인 패턴이 여러가지 있는 것 같은데 어떤 것이 있는 지,  
        각각 어떤 특장점이 있는 지 궁금하다.

- UIKit 클래스들을 다룰 때, 꼭 처리해야하는 어플리케이션 쓰레드 이름은 무엇인가?  
    - Main thread  
    - UIKit의 클래스들은 따로 적혀있지 않는 한 앱의 main thread 또는 main dispatch queue에서만 사용할 수 있다.  
        특히, UIResponder을 상속하거나, 앱의 UI를 조정하는 클래스에 적용된다.

    - UIKit을 메인 스레드에서 사용해야 하는 이유는 여러가지가 있는데 특히,  
        1. UIKit 내의 모든 프로퍼티를 thread-safe하게 만들어주면, 비현실적으로 프레임워크가 커진다.  
        2. UI에 관여하기 때문에, 비동기적으로 각각의 스레드가 뷰를 처리하면 제각각의 순서대로 뷰가 로드, 업데이트 되며, 3번 스레드에서 셀을 없앴는데, 4번 스레드에서 셀을 사용하는 등의 문제가 발생할 수 있다.  

- TableView의 동작 방식과 화면에 Cell을 출력하기 위해 최소한 구현해야 하는 DataSource 메서드를 설명하시오.
    - 동작 방식
        - 테이블 뷰는 데이터 기반으로 작동한다.  
            개발자에 의해 정의된 데이터소스 객체로 부터 데이터를 받아오고  
            그 데이터소스 객체는 앱의 데이터를 관리하고, 테이블 뷰의 셀들을 정의하는 역할을 한다.  
        - 구현은 일반적으로 테이블 뷰가 들어가는 상위 객체에 TableView를 추가하고,  
            해당 객체가 UITableViewDataSource 프로토콜을 채택하게 한다.  
            그 후, numberOfRowInSection과 cellForRowAt 메서드를 정의하면  
            테이블 뷰와 각 셀에 들어간 데이터가 열로 출력된다.  
        - 테이블 뷰가 내려가면서 새로운 셀을 반환하는 것은 cellForRowAt 메서드에서  
            일반적으로 반환할 셀을 정의할 때, TableView의 dequeReusableCell 메서드를 사용하는데,  
            셀이 화면 밖으로 밀려나면 reusableQueue에 들어가고,   
            다시 나오는 셀은 reusableQueue에서 나오게 되면서 모든 셀을 만드는 것이 아닌 셀을 재사용한다.  
            (모든 셀을 만드는 것은 앱 성능 측면에서 비효율적임)  
  
    - DataSource 메서드

        ```swift
        // 1
        // 섹션에 표시할 열의 개수를 반환하는 메서드
        func tableview(_ tableView: UITableView, numberOfRowInSection section: Int) → Int

        // 2
        // 각 열에 들어갈 cell을 반환하는 메서드
        func tableview(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell
        ```
