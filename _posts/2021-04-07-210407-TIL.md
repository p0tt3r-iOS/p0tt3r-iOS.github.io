```swift
    func generateCaption() -> String {
        let str = "캡션 내용"
        randomNum = Int.random(in: 10..<str.count)
        return str[0..<randomNum]
    }

    func generateDate() -> Date {
        randomNum = Int.random(in: 0..<10000)
        return Date().advanced(by: TimeInterval(-randomNum))
    }
```

## Question

---

- Cell Identifier를 파스칼 케이스로 사용하는 게 맞을까, 카멜 케이스로 사용하는 게 맞을까?  
    (정답이 없다면 어떤 방법이 사용하기 편할까?)
    > Follow Cocoa's identifier convention  
    Both Swift and Objective C use camelCase for identifiers.  
    Names never contain _ or - characters.  
    With the exception of types, classes & protocols, all names should have a lower case first letter.  
    Name your segues in camelCase and give them a lower case first letter.  
    위 내용을 보면, 타입, 클래스, 프로토콜 외의 모든 이름은 camelCase를 사용하라 되어있으니,
    카멜 케이스를 사용하는 것이 맞는 것 같다.
    
    [Stackoverflow: Storyboard Segue Identifier naming conventions](https://stackoverflow.com/questions/15370133/storyboard-segue-identifier-naming-conventions)
