### didSet으로 클래스 내의 다른 프로퍼티의 값을 지정하는 것과 { }의 차이?  

단순히 말로 설명하기 어려운 부분이 있어서 아래 코드로 두 케이스를 보이려 한다.

1. { } (때때로, 등호로 값을 주는 것이 아닌, 클로저로 값을 지정하는 경우가 있다.)
이 케이스는 이해가 부족하니 케이스 자체를 공부해야 할 것 같다.

    ```swift
    class TemperatureUnit {
        enum Unit {
            case celcius, fahrenheit
        }
        
        static let shared  = TemperatureUnit(unit: .celcius)
        
        var unit: Unit
        var boolValue: Bool {
            return self.unit == Unit.celcius ? true : false
        }
        
        private init(unit: Unit) {
            self.unit = unit
        }
    }
    ```

2. 아래 케이스가 문제인데, didSet을 통해 다른 프로퍼티의 값을 지정하려 하면 값이 비어있다고 에러가 발생한다.

    ```swift
    class TemperatureUnit {
        enum Unit {
            case celcius, fahrenheit
        }
        
        static let shared  = TemperatureUnit(unit: .celcius)
        
        var unit: Unit {
            didSet {
                boolValue = self.unit == Unit.celcius ? true : false
            }
        }
        var boolValue: Bool
        
        private init(unit: Unit) {
            self.unit = unit
        }
    }
    ```

    첫 번째 케이스는 생성될 때, boolValue가 리턴 값을 통해 값을 초기화 하는 것이고,  
    두 번째 케이스는 unit이 초기화 된 후에 boolValue 값을 초기화 해주면서 시차 떄문에   
    위와 같은 에러가 발생하나 싶은데, 다른 케이스를 통해 공부해봐야겠다.
