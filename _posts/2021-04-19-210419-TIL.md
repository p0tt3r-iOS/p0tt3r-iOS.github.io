- Bound와 Frame의 차이점
    - 나의 답  
        기본적인 개념부터 정리하자면, Frame은 SuperView 기준에서의 View의 위치와 크기이다.  
        Bound는 그와 다르게 자신 기준에서 위치와 크기를 나타낸다.  
        예시로 Superview가 300 * 500인 상황에서, 20 * 20 크기의 Subview가 (100, 100)에 위치한다면  
        Frame은 100, 100, 20, 20이지만, Bound는 subview 자신을 기준으로 삼기 때문에 (0, 0, 20, 20)이다. 
  
    - 찾은 답  
        둘의 정의에서 차이를 보이는데, Frame은 SuperView의 좌표시스템 안에서  
        View의 위치와 크기를 CGRect 타입으로 나타내고,  
        Bounds는 자신만의 좌표시스템 안에서 View의 위치와 크기를 CGRect 타입으로 나타낸다.  
        예시로 Superview 기준에서 20 * 20 크기의 Subview가 (100, 100)에 위치한다면  
        Frame은 100, 100, 20, 20이지만, Bounds는 자신의 좌표시스템을 기준으로 0, 0, 20, 20이다.  
  
        Bounds를 변경하는 것은 해당 위치에서 View를 다시 그리라는 의미이다.  
            (그래서 자신은 움직이지 않지만, 자신의 Subview는 움직이는 것처럼 보인다.)

- 실제 디바이스가 없을 경우, 개발 환경에서 할 수 있는 것과 없는 것울 설명하시오.
    - 찾은 답
        - 할 수 있는 것  
            1. 디버그 메뉴에서 디버그 툴을 사용할 수 있다.  
            2. UI가 실제 디바이스보다 빠르고 부드럽게 작동할 수 있다.  
            3. 앱에서 iCloud를 사용하는 경우, iCloud 연동을 테스트 할 수 있다.  
            4. 크래시 로그를 콘솔에서 확인할 수 있다.  
            5. 앱의 기본적인 기능(실행, 화면 전환 등) 테스트  
            6. UI Layout 테스트  
            
        - 할 수 없는 것  
            1. 실제 기기의 하드웨어가 필요한 기능인 조도센서, 기압센서, 근접센서, 오디오 / 비디오 인풋(카메라 / 마이크)을 이용한 테스트  
                (FaceID는 얼굴 인식은 안되지만, 인식 여부 처리는 시뮬레이터 메뉴에서 가능)  
            2. 앱 성능, 메모리 사용량, 네트워크 속도 등을 정확하게 테스트할 수 없다.  
            3. 시뮬레이터에서 지원하지 않는 API가 있기 때문에, 애플 푸시 알림을 받거나 보낼 수 없다.  
            4. 사진, 달력, 연락처, 리마인더에 접근하기 위한 개인정보보호 알림  
            5. UIBackgroundModes 키  
            6. Handoff support  
            7. External Accessory, IOSurface, Media Player, Message UI, UIVideoEditorController 프레임워크는 지원하지 않음  
            8. iPad mini 디바이스를 테스트할 수 없습니다.  
  
        참고자료  
        [Apple developer documentation: Testing and Debugging in Simulator](https://developer.apple.com/library/archive/documentation/IDEs/Conceptual/iOS_Simulator_Guide/TestingontheiOSSimulator/TestingontheiOSSimulator.html)

- 앱이 foreground / background에 있을 때, 어떤 제약사항이 있나요?
    - 나의 답  
        - background: UI에 관여할 수 없다.
  
    - 찾은 답  
        - Foreground  
            메모리 및 기타 시스템 리소스에 높은 우선 순위를 가지며 시스템은 이러한 리소스를 사용할 수 있도록 필요에 따라 background 앱을 종료한다.  
        - Background  
            가능한 작은 메모리 공간을 사용해야 한다. 백그라운드에서 카메라나 시스템 데이터베이스와 같은 자원에 접근하면 시스템이 앱을 종료 시킬 것이다.

- Decimal(10진수)를 Binary(2진수)로 바꾸는 코드  

    ```swift
    let decimal = 13
    var binary = String(decimal, radix: 2)
    print(binary) // "1101"
    ```
